# Модуль Балаково (Терминал)

## Обзор
Модуль **Balakovo** предназначен для детального планирования работы грузового терминала. Он фокусируется на управлении причалами, очередью судов и ограничениями инфраструктуры.

## Основные функции

### 1. Планировщик причалов (`balakovo_planner.py`)
*   **Распределение слотов**: Назначает суда на конкретные причалы с учетом их характеристик (длина, осадка).
*   **Очередь**: Управляет очередностью постановки судов (FIFO или по приоритету).
*   **Оптимизация**: Пытается минимизировать общее время ожидания на рейде.

### 2. Проверка ограничений (`berth_constraints.py`)
Модуль проверяет множество физических и эксплуатационных ограничений:
*   **Длина причала (LOA)**: Судно не должно превышать длину причала.
*   **Осадка (Draft)**: Глубина у причала должна быть достаточной.
*   **Тип груза**: Причал должен поддерживать тип груза (наливной/насыпной).
*   **Ремонтные окна**: Учет периодов вывода причалов из эксплуатации.

### 3. Отчетность (`balakovo_gantt.py`, `balakovo_report.py`)
*   График занятости причалов (Berth Occupancy Chart).
*   Отчет об утилизации (Utilization Report): % времени занятости каждого причала.

## Входные данные
*   `berths_balakovo.csv`: Характеристики причалов (длина, глубина, тип).
*   `fleet_balakovo.csv`: Суда, планируемые к погрузке.
*   `cargo_plan_balakovo.csv`: План отгрузок.
*   `restrictions_balakovo.csv`: Временные ограничения (ремонты, дноуглубление).

## Особенности
Модуль поддерживает "Тетрис-планирование", позволяя размещать несколько небольших судов на одном длинном причале, если это позволяют правила безопасности.
